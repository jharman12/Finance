# Goal

Extend this app to a full AI home assistent application. This would entail adding calenders, to-do-list, shopping lists, allowing emailing, and configuring the local LLM to be an AI interface between them.

# Calendar

This should be a google calendar like feature. This should be a clean looking interface that allows for multiple "Calendars" where you can add events under different calendars such as "Jacks Calendar", "Work Calendar", "Family Calendar", etc .... 

The Ai should be able to see what events are scheduled and add new events.

# To-Do List

Should allow for customization per person by day.

# shopping List

This should be a house hold list that can add and track groceries list. It should track previous lists and suggest items based on trends (i.e. ive bought milk once a week every week so suggest milk).

# emailing

AI should be able to either email you or text you things that you have asked. Example.) heres your to-do list today and events on your calendar.

# AI Interface

I think I would like to have the AI modified through agents using local systems. I would like to have agents associated with each feature in the application (Finance, Calendar, etc...) and an agent to manage the other agents. These should remember past conversations and be customizable, allowing the users to tune their agents to their liking and create more or completely different agent systems.

I would also like to allow for online AI like Claude to optionally work as the AI as well.

# Remote Connection

Remote connection currently doesnt allow for changing of the wake word. It should not be hard coded and should replicate the current work word for the main application.

Remote connections should be lasting. After pairing a device, opening and closing the application should remember the device was connected and try to connect if the remote device is running.

Remote connection should be wired directly into the AI text input

# Text to Speech

When recieving tasks from a remote device, the agent should respond in a concise answer and be speak the response back on that device as if it was a normal conversation.
